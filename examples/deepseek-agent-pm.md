# Example: DeepSeek AI / Agent Product Manager

This example shows the kind of output the skill should produce from an AI / Agent PM job description.

## 1. Role Snapshot

- Company / team: DeepSeek product team
- Role: AI Product Manager / Agent Harness Product Manager
- Product domain: Chat, App, open platform, Agent Harness, AI Coding, workflow products
- User type: general users, developers, researchers, open-source community
- Seniority signal: 2+ years PM experience, full-cycle delivery, strong autonomy
- Core hiring thesis: can turn frontier model capability into usable, measurable, low-friction AI product experiences

## 2. Inferred Screening Mechanism

### Hard Filters

- Product experience or complete delivery experience from requirement to launch
- AI / conversation / Agent product exposure
- Strong user insight, product taste, and data sense
- Ability to work with model, algorithm, engineering, and operation teams
- For Agent Harness: understanding of Agent mechanisms and strong hands-on AI tool usage

### ATS Keyword Filters

- Product keywords: AI product, Agent, Agent Harness, Chat, App, open platform, workflow, AI Coding
- Technical keywords: LLM API, Agent Loop, Tool Use, Reasoning, Planning, Skills, MCP, Memory, Subagent, Multi-Agent
- Methodology keywords: Prompt Engineering, Context Engineering, Harness Engineering, UI/UX, Roadmap, PRD
- Metrics keywords: retention, conversion, stickiness, task completion, A/B test, funnel analysis, gray release
- Stakeholder keywords: researchers, model team, algorithm team, engineering, operations, open-source community

### Human Evaluation Criteria

- Product judgment: can distinguish real user value from model novelty
- Technical taste: understands model behavior and Agent failure modes
- User insight: can reduce usage threshold and improve task completion
- Data rigor: can combine qualitative feedback with behavioral data
- Execution: can independently push complex projects from research to launch

## 4. Resume Module Strategy

### Profile / Summary

Must signal:

- AI Agent PM positioning
- 0-1 product ownership
- Skill Market / AI knowledge platform / workflow experience
- Data-driven product judgment

Suggested wording:

> 3 years of product experience across complex interaction systems, growth, and AI Agent product 0-1. Focused on Agent Template, Skill Market, AI knowledge base, and workflow products, with experience turning model capabilities into low-friction, configurable, and measurable product experiences.

### Core Skills / Keywords

- AI Agent: Agent Template, Agent Harness, Tool Use, Skills, MCP, Prompt / Context Engineering
- Product: 0-1 Roadmap, PRD, user research, competitor analysis, UI/UX, prototype
- Data: event tracking, funnel analysis, retention, conversion, A/B test, gray release
- Technical fluency: AI Coding, Unity, C++, API, workflow automation

### Work Experience

#### CodeBanana

- Target positioning: strongest match for Agent Harness and Skill Market
- Keywords to include: Agent Template, Skill Market, workflow, developer experience, platform governance
- Evidence to show: user-side skill usage, developer-side skill publishing, backend management, organization management

Suggested bullets:

- Owned Agent Template and Skill Market modules, positioning Agent as a container for intelligent workflows and reducing the path from task intent to executable workflow.
- Designed embedded Skill Market flows for users to discover, add, and invoke skills inside Agent scenarios, expanding Agent from fixed templates into configurable workflows.
- Owned Skill Market platform and admin workflows, including skill publishing, review, categorization, state management, permissions, versioning, and data feedback.

#### TicNote

- Target positioning: AI product 0-1 and model capability productization
- Keywords to include: AI knowledge base, 0-1 Roadmap, cross-file query, structured generation, model evaluation
- Evidence to show: online iteration, competitor analysis, user data, funnel metrics, algorithm collaboration

Suggested bullets:

- Led TicNote software product from 0-1 to online roadmap, evolving a hardware recorder from an information entry point into an AI knowledge platform.
- Defined the core flow of recording assets, transcription, summarization, knowledge base, and AI CRUD, supporting cross-file query and structured content generation.
- Used competitor analysis, user feedback, and product funnels to define iteration priorities; worked with algorithm teams on domain vocabulary recall, speaker segmentation, and multilingual alignment.

## 5. Evidence Gap Checklist

### Can Claim If True

- Deep use of Claude Code, Codex, Cursor, GitHub Copilot, Manus
- Prompt / Context Engineering practice
- AI-assisted prototype or UI design

### Need Evidence

- Concrete Skill Market metrics: install conversion, invocation rate, repeat use, developer supply
- Concrete TicNote funnel metrics: transcription completion, content consumption, retention
- User research examples: interview count, survey structure, insight extraction

### Do Not Fake

- Researcher collaboration if only worked with product / algorithm teams
- Open-source community maintenance if there was no real community work
- Advanced LLM infra concepts unless you can explain them in interview

## 6. Likely Interview Probes

- How do you judge whether an Agent is actually helping users?
- How should a Skill Market balance developer experience and platform governance?
- What are the main failure modes in Agent products?
- How do you combine user feedback and behavioral data to prioritize product iterations?
- What differences have you observed between Claude Code, Codex, Cursor, and Copilot?
- How would you define metrics for Agent task success?

> From complex interaction systems to growth and AI Agent product 0-1, able to connect model capability, developer ecosystem, and user scenarios into usable product experiences.

2. TicNote: AI knowledge platform
3. DupDub / growth: data and commercialization
